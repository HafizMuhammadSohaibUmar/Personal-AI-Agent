1) Overview

This project runs a local web server that exposes:

  • Custom web chat UI at /
  • LangServe API at /chat/* (e.g. /chat/invoke)
  • LangGraph workflow that calls tools to manage tasks and calendar events stored locally in SQLite

The assistant is designed to keep scheduling local-only and to propose scheduling decisions based on stored priority rules and working-hour constraints.

2) Features & automation

2.1 Task automation (local-only)

  • Create tasks (title, due date, tags, priority, estimate)
  • List tasks (open/completed)
  • Complete tasks

2.2 Calendar + scheduling automation (local-only)

  • Create and list events stored in SQLite
  • Find free slots within constraints
  • Auto-schedule when user does not specify an exact time

2.3 Daily planning + time-blocking

  • Generate a daily plan using current tasks, events, and priority rules
  • Time-block top tasks into free time within 09:00–21:00 local time

2.4 Not implemented (yet)

  • No background reminders or push notifications
  • No email/Google Calendar integrations
  • No autonomous rescheduling unless you ask for it

3) Quickstart (Mode B / Web UI)

Run from the project root:

poetry run python app.py

Then open:

  • UI: http://localhost:8000/
  • API docs: http://localhost:8000/docs
  • LangServe playground: http://localhost:8000/chat/playground/

Tip: in the UI, click Reset to reset browser session state.

4) Environment variables

Set one of these environment variables:

  • MISTRALAI_API_KEY (preferred)
  • MISTRAL_API_KEY

The assistant code maps MISTRALAI_API_KEY to MISTRAL_API_KEY when needed.

5) Web UI & API endpoints

5.1 Custom UI

  • GET / returns HTML/CSS/JS chat UI

5.2 LangServe API

  • POST /chat/invoke is called by the UI

Example request:

{
  "input": {
    "messages": [
      {"type": "human", "content": "List my tasks"}
    ]
  },
  "config": {
    "configurable": {
      "thread_id": "browser-session-id"
    }
  }
}

6) Architecture (LangGraph)

Main graph: support_agent/personal_assistant/graph.py

  • assistant node: LLM decides what to do and produces tool calls
  • tools node: executes tools and returns tool results

Routing rule:

  • If the assistant output contains tool calls → execute tools → return to assistant
  • Else finish

7) Storage (SQLite)

  • Assistant data: tasks/events/preferences stored locally via support_agent/personal_assistant/db.py
  • Graph checkpoints: stored in checkpoints_pa.sqlite (LangGraph checkpointer)

8) Tools catalog

Tools live in: support_agent/personal_assistant/tools.py

Tasks

  • create_task(...)
  • list_tasks(user_id, status="open")
  • complete_task(user_id, task_id)

Priority rules

  • get_priority_rules(user_id)
  • set_priority_rules(user_id, rules_json)

Events + scheduling

  • create_event(...)
  • list_events(...)
  • find_free_slots(...)
  • auto_schedule_event(...)

Planning

  • daily_plan(user_id, day, max_tasks=...)
  • timeblock_top_tasks(user_id, day, ...)

10) Troubleshooting

10.1 Use the correct environment

Always start the server with:

poetry run python app.py

10.2 HTTP 500 errors

Check server logs. Common causes:

  • Missing MISTRALAI_API_KEY
  • Wrong Python environment or dependency mismatch
  • Provider strictness around tool-call sequencing

10.3 Reset conversation state

  • Use UI Reset to generate a new browser session thread id
